# 创建项目

```bash
npm add -D vitepress@2.0.0-alpha.16
```
将 `2.0.0-alpha.16` 替换成 `next` 可以获取最新版本。

# 初始化

```bash
npx vitepress init
```

根据向导初始化项目（按默认直接回车即可）。
```bash
T  Welcome to VitePress!
|
o  Where should VitePress initialize the config?
|  ./
|
o  Where should VitePress look for your markdown files?
|  ./
|
o  Site title:
|  My Awesome Project
|
o  Site description:
|  A VitePress Site
|
o  Theme:
|  Default Theme
|
o  Use TypeScript for config and theme files?
|  Yes
|
o  Add VitePress npm scripts to package.json?
|  Yes
|
o  Add a prefix for VitePress npm scripts?
|  Yes
|
o  Prefix for VitePress npm scripts:
|  docs
|
—  Done! Now run npm run docs:dev and start writing.
```
创建完成提示：`Done! Now run npm run docs:dev and start writing.`

 - 启动项目： `npm run docs:dev` 
 - 编译项目：`npm run docs:build` 
 - 监听项目：`npm run docs:serve` 

# 项目结构

认识项目结构：
```bash
st-blog                 # 项目根目录
├─ .vitepress
│  ├─ cache             # 这里面的文件可忽略
│  └─ config.mts        # 全局配置文件
├─ api-examples.md
├─ index.md             # 主页
├─ markdown-examples.md
├─ package-lock.json
└─ package.json
```

为了方便笔记文件集中管理，所以按如下修改项目结构：
```bash
st-blog 
├─ .vitepress
│  ├─ cache
│  └─ config.mts            # 全局配置
├─ docs                     # 专门放md文件
│  ├─ api-examples.md
│  └─ markdown-examples.md
├─ public
│  ├─ favicon.ico           # 网站的图标
│  └─ logo.png              # 网站的logo
├─ index.md                 # 网站的首页
├─ package-lock.json
├─ package.json
└─ README.md
```
`api-examples.md`，`markdown-examples.md` 是项目创建自带的示例文件，可删除。
