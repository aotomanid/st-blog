# 部署网站

为了方便随时查看网站，就要提到部署了。[官方](https://vitepress.dev/zh/guide/deploy#build-and-test-locally)给出很多部署方案。

以下采用的是 [GitHub Pages](https://pages.github.com/)

# 准备工作

开始前，确保电脑中有[Git](https://git-scm.com/)，并有一个[GitHub](https://github.com/)账号。

登录[GitHub](https://www.github.com)，点击右上角的 `+` 号，选择 `New repository`。
输入仓库名称，并选择是否公开（免费用户只能公开仓库），然后点击 `Create repository` 创建仓库。

创建完成后，复制 `SSH` 地址

# 创建仓库

在项目的根目录执行git命令：

```bash
git init
```

根目录创建 `.gitignore` 文件并添加以下内容：

```text
node_modules/
```

提交会忽略文件夹，避免将 node_modules 文件夹上传到 GitHub。

# 推送远程仓库

分步执行 git 命令：

```bash
git add . # 添加所有文件
git commit -m "init" # 提交
git remote add origin 换成复制的 SSH 地址 # 关联远程仓库
git push -u origin main # 推送到远程仓库
```

> [!TIP] 提示
> 如遇到两端不同步之类的报错，确保远程仓库可以覆盖的情况下，执行强制推送：
>  `git push -u origin main --force` 。

# 开启 GitHub Pages

开启 Github Pages 之前需要修改几个配置([官方说明](https://vitepress.dev/zh/guide/deploy#build-and-test-locally))，总结如下：

1. 原有的 `"docs:build": "vitepress build"` 改为 `"docs:build": "vitepress build docs"`；
   
    因为之前改了目录结构，加docs文件依赖会产生错误。我将输出方式放到了 `config.mts` 中，这里就不改了。

2. 在 `config.mts` 文件中添加 `base` 和 `outDir` 配置;
   
```mts
export default defineConfig({
    base: '/st-blog/', // 匹配路径
    outDir: 'docs/.vitepress/dist', // build 输出位置
...
})
```

进入项目的设置页面，找到 `GitHub Pages` 选项，将 `Source` 的 `Deploy from a branch` 改为 `GitHub Actions` ，点击 `GitHub Pages Jekyll` 的 `configure`。

在弹出的页面中修改名称，`jekyll-gh-pages.yml` => `deploy.yml` 。并覆盖掉内容：

```yml
# 构建 VitePress 站点并将其部署到 GitHub Pages 的示例工作流程
#
name: Deploy VitePress site to Pages

on:
  # 在针对 `main` 分支的推送上运行。如果你
  # 使用 `master` 分支作为默认分支，请将其更改为 `master`
  push:
    branches: [main]
    paths:
     - 'docs/**'

  # 允许你从 Actions 选项卡手动运行此工作流程
  workflow_dispatch:

# 设置 GITHUB_TOKEN 的权限，以允许部署到 GitHub Pages
permissions:
  contents: read
  pages: write
  id-token: write

# 只允许同时进行一次部署，跳过正在运行和最新队列之间的运行队列
# 但是，不要取消正在进行的运行，因为我们希望允许这些生产部署完成
concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  # 构建工作
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v5
        with:
          fetch-depth: 0 # 如果未启用 lastUpdated，则不需要
      # - uses: pnpm/action-setup@v4 # 如果使用 pnpm，请取消此区域注释
      #   with:
      #     version: 9
      # - uses: oven-sh/setup-bun@v1 # 如果使用 Bun，请取消注释
      - name: Setup Node
        uses: actions/setup-node@v6
        with:
          node-version: 24
          cache: npm # 或 pnpm / yarn
      - name: Setup Pages
        uses: actions/configure-pages@v4
      - name: Install dependencies
        run: npm ci # 或 pnpm install / yarn install / bun install
      - name: Build with VitePress
        run: npm run docs:build # 或 pnpm docs:build / yarn docs:build / bun run docs:build
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist

  # 部署工作
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    needs: build
    runs-on: ubuntu-latest
    name: Deploy
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

定时构建：
```yaml
on:
  schedule:
    # 每5分钟运行一次
    - cron: "*/5 * * * *"
    # 每天UTC时间0点运行一次
    - cron: "0 0 * * *"
```

仅当 docs 文件夹有更新时才构建
```yaml
on:
  push:
    branches: [ main ]
    paths:
      - 'docs/**'
      - '!src/README.md'  # 排除特定文件
```

右上角 `Commit changes` 保存。

进入 `Actions` 页面，等待构建完成。

访问 `https://用户名.github.io/仓库名/` 就可以看到网站了。
