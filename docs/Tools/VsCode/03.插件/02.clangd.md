# 必要插件

> [!TIP] 在 VSCode 中，这三个插件通常配合使用，为 C/C++ 项目提供完整的开发体验：
> 用 clangd 方案替代微软官方的 C/C++ ，主要原因是官方的插件会卡顿，而且日常使用时，跳转不是那么准确，代码补全之类的提示来的卡顿。

## clangd
- 基于 Clang 的语言服务器，提供智能的代码补全、语法高亮、实时错误检查、符号跳转、重命名等语言服务。它通过解析编译数据库（如 `compile_commands.json`）来理解项目结构，性能优秀且准确。
- 安装此插件时，会提示没有发现语言服务器，是否直接安装，直接关掉（后续自己手动配置）。

> [!CAUTION] 此插件与官方 C/C++ 插件互斥，如果安装了 C/C++ 要禁用或者卸载。

## CMake Tools
- 用于集成 CMake 构建系统的插件。它能自动配置 CMake 项目、选择工具链、运行构建、执行测试，并支持在 VSCode 中直接切换构建类型（Debug/Release）和目标。通常与 clangd 配合时，会生成编译数据库供 clangd 使用。

## CodeLLDB
- 一个基于 LLDB 的调试器前端，提供强大的调试功能。支持设置断点、单步执行、查看变量/寄存器、评估表达式，并能处理多线程、嵌入式等多种复杂场景。它与 VSCode 的调试界面深度集成，是 C/C++ 调试的常用选择。
- 安装需要翻墙，如果下载失败，可能是网络问题。

# 配置环境
- 下载[整合包](https://winlibs.com)，选择前面的下载，后面的是不包含 `LLVM/Clang/LLD/LLDB` 的包。
- `ctrl+f` 搜索 `GCC 14.2.0 (with POSIX threads)` 选择 `release 3` 版本，Win64 选择 `mingw64` ，win32 选择 `mingw32`

> [!IMPORTANT] 不要选择 without 后面的包。

## 解压

- 解压包后得到 `mingw32/64` 文件夹，将其放入 `自定义` 路径中；
- 我常用的放置路径：`D:\software\sdk` 

## 环境变量

- 将路径 `自定义路径\mingw64\bin` 添加到 `系统环境变量` 中；
> [!CAUTION] 注意是系统环境变量。

## 验证

- 输入命令查看有版本号输出说明成功；
```shell
gcc --version
cmake --version
clangd --version	
```

## 避坑指南
- 插件设置中的 `path` 如果带有路径，大概率没法用。

# 怎么用？
1. vs code 打开一个空文件夹；
2. `ctrl+shift+p` 输入 `cmake:quick start` （快速入门）；
3. 输入可执行文件的名称，作用如例：输入 `hello` => 最终 `hello.exe`；
4. 按项目需要选择 `C` 或者 `C++` ；
5. `Executable` 创建可执行文件；
6. `CTest` & `CPack` 都不选；
7. 添加新预设 => 从编译器创建；
8. 选择 `clang`，如果没有clang，先执行扫描编译器，还是没有检查是否正常安装
9. 输入 `build` 后的文件夹名称，如：`lang` => ./out/build/`lang`/hello.exe
## 运行和调试
1. vs code 左下角【生成】可执行文件，即 `.exe`；
2. 打开【运行和调试】；
3. 创建 `launch.json` 文件；
4. 修改 `configurations` 参数；
```json
{
    // 使用 IntelliSense 了解相关属性。 
    // 悬停以查看现有属性的描述。
    // 欲了解更多信息，请访问: https://go.microsoft.com/fwlink/?linkid=830387
    "version": "0.2.0",
    "configurations": [
        {
            "type": "lldb",
            "request": "launch",
            "name": "Launch",
            "program": "${workspaceFolder}/out/build/lang/hello.exe",
            "args": [],
            "cwd": "${workspaceFolder}/out/build/lang"
        }
    ]
}
```
