# SSH 密钥

SSH 密钥是一种用于验证用户身份的加密密钥。它不是必须的，但是推荐使用的。
GitHub 默认会要求用户在连接自己仓库的时候使用 SSH 密钥。

# 查看密钥

先查看本地是否有密钥：
```bash
# 新环境大概率会报错 ，因为这个目录不存在
cd ~/.ssh
```
如果提示找不到，则执行以下命令：
```bash
ssh-keygen -t rsa -C 你的邮箱
```

然后一直按回车，直到生成密钥（出现一个方块[SHA256]就是成功了）。

# 获取密钥

```bash
# 进入ssh目录
cd ~/.ssh
# 查看ssh 公钥  进行复制
cat id_rsa.pub
```
# 添加密钥

进入 `GitHub` 账号的设置页面，找到 `SSH and GPG keys` 。
点击 `New SSH key` ，将生成的密钥粘贴到 `Key` 框中，并输入名字，名字可以随意，但自己要能分辨 。

# 测试密钥

```bash
ssh -T git@github.com
```

能看到 `Hi username! You've successfully authenticated, but GitHub does not provide shell access.` 就说明能正常连通。
